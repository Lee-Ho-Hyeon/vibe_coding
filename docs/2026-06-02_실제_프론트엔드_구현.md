# 2026-06-02 실제 프론트엔드 구현

## 프론트엔드 기본 구조 구현

이전 단계에서 구현한 FastAPI 기반 백엔드와 SQLite 데이터베이스를 실제 사용자가 활용할 수 있도록 프론트엔드를 구현하였다.

프론트엔드는 HTML, CSS, JavaScript를 사용하여 개발하였으며, 사용자가 별도의 API 호출 과정 없이 웹 브라우저에서 선수 정보를 검색할 수 있도록 구성하였다.

전체 프론트엔드는 다음 세 파일로 구성하였다.

- `index.html`
- `style.css`
- `app.js`

---

## index.html 구현

프론트엔드의 기본 화면은 `index.html`을 통해 구현하였다.

사용자가 다양한 검색 기능을 사용할 수 있도록 다음 구성 요소를 배치하였다.

- 프로젝트 제목
- 검색 유형 선택 메뉴
- 검색어 입력창
- 검색 버튼
- 클럽 목록 조회 버튼
- 검색 결과 표시 영역
- 선수 상세 정보 표시 영역

특히 검색 유형 선택 메뉴를 통해 사용자가 다음 세 가지 검색 방식을 선택할 수 있도록 구현하였다.

- 선수 검색
- 국적 검색
- 클럽 검색

이를 통해 하나의 검색창으로 다양한 검색 기능을 수행할 수 있도록 구성하였다.

---

## style.css 구현

사용자 편의성을 높이기 위해 CSS를 활용하여 화면 디자인을 적용하였다.

주요 적용 내용은 다음과 같다.

- 반응형 화면 구성
- 검색창 및 버튼 디자인
- 검색 결과 카드 형태 출력
- 마우스 오버 효과 적용
- 자동완성 결과 박스 구성

또한 검색 결과와 선수 상세 정보를 각각 독립된 영역에 표시하여 사용자가 결과를 쉽게 확인할 수 있도록 하였다.

---

## JavaScript 기능 구현

검색 기능은 `app.js`를 통해 구현하였다. :contentReference[oaicite:0]{index=0}

초기에는 단순히 선수 검색 기능만 구현할 계획이었으나, 개발을 진행하면서 기능을 하나씩 추가하는 방식으로 프론트엔드를 발전시켰다.

---

### 선수 검색 기능 구현

가장 먼저 구현한 기능은 선수 검색 기능이었다.

사용자가 선수 이름 또는 별칭(alias)을 입력하고 검색 버튼을 누르면 백엔드의 선수 검색 API를 호출하도록 구현하였다.

검색 결과는 화면에 목록 형태로 출력되며, 사용자가 원하는 선수를 선택할 수 있도록 구성하였다.

---

### 자동 추천 검색 기능 추가

이후 사용자가 선수 이름 전체를 입력하지 않아도 검색할 수 있도록 자동 추천 기능을 추가하였다.

사용자가 검색창에 입력을 시작하면 일정 시간 이후 자동으로 검색 API를 호출하도록 구현하였다. :contentReference[oaicite:1]{index=1}

이를 통해 사용자가 선수 이름 일부만 입력하더라도 관련 선수 후보를 즉시 확인할 수 있도록 개선하였다.

예를 들어 다음과 같은 검색이 가능하도록 구현하였다.

```text
호날
→ 크리스티아누 호날두

메시
→ 리오넬 메시
```

---

### 선수 상세 정보 조회 기능 구현

검색 결과에서 특정 선수를 선택하면 해당 선수의 상세 정보를 조회할 수 있도록 구현하였다. :contentReference[oaicite:2]{index=2}

상세 정보에는 다음 항목을 표시하도록 구성하였다.

- 선수 이름
- 국적 정보
- 클럽 경력

이를 통해 사용자가 검색 결과를 다시 확인하지 않아도 선수 정보를 한눈에 확인할 수 있도록 하였다.

---

### 국적 검색 기능 추가

이후 국적 검색 기능을 추가하였다.

사용자가 특정 국가를 입력하면 해당 국가를 후보 목록으로 보여주고, 국가를 선택하면 해당 국적을 가진 선수 목록을 조회하도록 구현하였다. :contentReference[oaicite:3]{index=3}

예를 들어 다음과 같은 검색이 가능하다.

```text
대한민국
→ 손흥민
→ 김민재
→ 이강인
```

---

### 클럽 검색 기능 추가

클럽 검색 기능도 동일한 방식으로 구현하였다.

사용자가 클럽명을 입력하면 관련 클럽 후보를 표시하고, 특정 클럽을 선택하면 해당 클럽 경력을 가진 선수 목록을 출력하도록 구현하였다. :contentReference[oaicite:4]{index=4}

이를 통해 사용자가 클럽명을 기준으로 선수들을 탐색할 수 있도록 하였다.

---

### 클럽 목록 조회 기능 추가

사용자가 정확한 클럽명을 모를 수 있다는 점을 고려하여 별도의 클럽 목록 조회 기능을 추가하였다. :contentReference[oaicite:5]{index=5}

사용자가 "클럽 목록 보기" 버튼을 클릭하면 DB에 저장된 모든 클럽을 화면에 표시하도록 구현하였다.

이를 통해 사용자가 직접 클럽명을 입력하지 않고도 원하는 클럽을 찾을 수 있도록 하였다.

---

### 클럽 선택 기반 선수 조회 기능 구현

클럽 목록에서 특정 클럽을 선택하면 해당 클럽에서 활동했던 선수 목록을 조회할 수 있도록 구현하였다. :contentReference[oaicite:6]{index=6}

예를 들어 사용자가 레알 마드리드를 선택하면 레알 마드리드 경력을 가진 선수 목록이 출력되며, 이후 특정 선수를 선택하면 상세 정보를 확인할 수 있도록 구성하였다.

프론트엔드 구현을 통해 다음 기능을 사용할 수 있게 되었다.

- 국적 검색

- 클럽 목록 조회
- 클럽 선택 기반 선수 조회

이를 통해 사용자는 웹 브라우저만으로 선수 정보를 쉽게 검색할 수 있게 되었다.

## 피드백 및 개선 방향

실제 구현 후 테스트를 진행한 결과 몇 가지 개선이 필요한 점을 발견하였다.

현재 구조에서는 선수 검색, 국적 검색, 클럽 검색, 선수 상세 정보, 클럽 목록 등의 기능이 모두 하나의 페이지에 표시되고 있다.

이로 인해 검색 결과가 많아질 경우 화면의 가독성이 크게 떨어지는 문제가 발생하였다.

특히 특정 클럽 경력을 가진 선수나 특정 국적의 선수는 수가 매우 많기 때문에 검색 결과가 길게 출력되며, 사용자가 원하는 정보를 찾기 어려운 경우가 있었다.

따라서 향후에는 검색 결과와 상세 정보를 별도의 페이지로 분리하고, 사용자가 검색 결과를 선택하면 다음 페이지로 이동하여 상세 정보를 확인하는 방식으로 구조를 개선할 계획이다.
