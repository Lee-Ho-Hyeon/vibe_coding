# 2026-05-22 구현 및 수정

## 위키데이터 기반 클럽별 선수 수집 구현

이전 구현 과정에서 선수 이름 기반 검색만으로는 안정적인 선수 정보 매칭이 어렵다는 문제를 확인하였다. 이에 따라 이름 검색 방식 대신, 위키데이터(Wikidata)의 고유 식별 번호(QID)를 활용하는 방향으로 구현 방식을 수정하였다.  

위키데이터에서는 각 클럽과 선수마다 고유 식별 번호(QID)가 존재하였다. 예를 들어 레알 마드리드의 경우 `Q8682`, 바르셀로나의 경우 `Q7156`과 같은 식별 번호를 가지고 있었다.  

이에 따라 특정 클럽의 QID를 입력하면 해당 클럽에서 활동했던 선수들을 자동으로 수집하는 방식으로 구현 방향을 변경하였다. 이후 GPT를 활용하여 `organize_players_by_club.py` 코드를 생성하였으며, 해당 코드는 입력된 클럽 QID를 기준으로 위키데이터에 질의(Query)를 보내 선수 정보를 수집하도록 구현하였다.  

초기 구현 단계에서는 다음과 같은 방식으로 쿼리를 작성하였다.

```sparql
?player wdt:P106 wd:Q937857 .
?player wdt:P54 wd:Q8682 .
```

위 쿼리의 의미는 다음과 같다.

- `P106`: 직업(property)
- `Q937857`: 축구 선수(association football player)
- `P54`: 소속 팀(property)
- `Q8682`: 레알 마드리드

즉, “직업이 축구 선수이며, 소속팀 정보에 레알 마드리드가 포함된 선수”를 검색하는 방식이었다.  

또한 국적 정보 역시 함께 저장할 수 있도록 다음과 같은 선택적 쿼리를 추가하였다.

```sparql
OPTIONAL { ?player wdt:P27 ?country . }
```

이를 통해 가능한 경우 선수의 국적 정보도 함께 수집하도록 구현하였다.

## 구현 과정에서 발생한 문제점 및 쿼리 수정

하지만 실제 실행 과정에서 몇몇 유명 선수들이 결과에서 누락되는 문제가 발생하였다. 대표적으로 크리스티아누 호날두와 같이 실제로 레알 마드리드 경력이 존재하는 선수임에도 검색 결과에 포함되지 않는 현상이 확인되었다.  

원인을 분석한 결과, 기존 방식은 단순히 선수의 소속팀 값을 직접 조회하는 방식(`wdt:P54`)이었기 때문에, 시작일·종료일 등 추가 메타데이터가 포함된 소속 기록(statement)을 정상적으로 처리하지 못하는 경우가 존재하였다.  

이에 따라 쿼리 구조를 다음과 같이 수정하였다.

기존 방식:

```sparql
?player wdt:P54 wd:Q8682 .
```

수정 후 방식:

```sparql
?player p:P54 ?statement .
?statement ps:P54 wd:Q8682 .
```

기존 방식은 단순히 “소속팀 값이 레알 마드리드인지”를 직접 확인하는 구조였다면, 수정 후 방식은 “선수의 소속팀 기록(statement) 내부에 레알 마드리드 정보가 포함되어 있는지”를 확인하는 구조로 변경된 것이다.  

이를 통해 시작일, 종료일 등 추가 정보가 포함된 선수 기록까지 안정적으로 검색할 수 있게 되었으며, 이전에 누락되었던 선수들 역시 정상적으로 검색 결과에 포함되는 것을 확인하였다.  

수정 이후에는 `club_{클럽 식별번호}_players.csv` 파일이 정상적으로 생성되었으며, 클럽별 선수 데이터를 안정적으로 저장할 수 있게 되었다.

## 선수 이름 표기 문제 및 데이터 정제

하지만 또 다른 문제점이 존재하였다.  

실제 위키데이터에 저장된 선수 이름과 국내 축구 게임 유저들이 일반적으로 사용하는 이름이 서로 다른 경우가 존재하였다. 예를 들어 일부 선수는 위키데이터 기준으로 full name(본명)이 저장되어 있었으며, 사용자들이 일반적으로 사용하는 이름과 차이가 발생하였다. 또한 특정 선수는 full name 전체를 입력해야만 검색이 가능한 경우도 존재하였다.  

이를 해결하기 위해 선수 이름 정보를 다음과 같이 세 가지 형태로 분리하여 저장하도록 구조를 수정하였다.

- `wikidata_name`
  - 위키데이터에 저장된 원본 공식 이름

- `display_name`
  - 실제 사이트에서 사용자에게 표시할 이름
  - 필요 시 수동 보정(name correction) 적용

- `alias`
  - 별칭, 약칭, 영어 이름 등 검색 보조용 이름

예를 들어 위키데이터에는 “프란시스코 요렌테 헨토”로 저장되어 있더라도, 실제 사용자 화면에는 “마르코스 요렌테”로 표시되도록 수정할 수 있도록 구현하였다.  

최종적으로 `club_{클럽 식별번호}_players.csv` 파일에는 다음과 같은 형태로 데이터가 저장되도록 구성하였다.

```text
qid
wikidata_name
display_name
country
club_qid
alias
```

이를 통해 이후 검색 시스템 구현 단계에서 사용자들이 일반적으로 사용하는 이름 기준으로 선수 검색이 가능하도록 데이터 구조를 개선하였다.
