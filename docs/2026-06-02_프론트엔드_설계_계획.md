# 2026-06-02 프론트엔드 설계 계획

## 프론트엔드 개발 필요성

이전 단계에서 FastAPI 기반 백엔드 구현을 완료하였다. 백엔드는 선수 검색, 클럽 검색, 국적 검색, 클럽 목록 조회 등의 기능을 API 형태로 제공할 수 있게 되었다.

하지만 현재 상태에서는 API를 직접 호출해야만 결과를 확인할 수 있으며, 일반 사용자가 쉽게 사용할 수 있는 형태의 화면은 존재하지 않았다.

따라서 사용자가 웹 브라우저를 통해 선수 정보를 쉽게 검색할 수 있도록 프론트엔드를 구현하기로 하였다.

---

## 프론트엔드 목표

프론트엔드의 주요 목적은 사용자가 복잡한 API를 직접 사용할 필요 없이 검색창과 버튼만으로 원하는 선수 정보를 조회할 수 있도록 하는 것이다.

또한 선수 이름 검색뿐만 아니라 클럽 검색, 국적 검색, 클럽 목록 조회 기능까지 직관적으로 사용할 수 있도록 설계하기로 하였다.

---

## 화면 구성 논의

논의 결과 초기 화면에는 다음 요소들을 배치하기로 하였다.

- 프로젝트 제목
- 검색창
- 검색 버튼
- 검색 유형 선택 기능
- 검색 결과 표시 영역
- 선수 상세 정보 표시 영역

이를 통해 사용자가 하나의 화면에서 모든 기능을 이용할 수 있도록 설계하였다.

---

## 검색 기능 설계

### 선수 검색

사용자가 선수 이름 또는 별칭(alias)을 입력하면 해당 선수를 검색할 수 있도록 구성하기로 하였다.

예를 들어 다음과 같은 검색이 가능하도록 설계하였다.

- 호날두
- CR7
- Cristiano Ronaldo

검색 결과에는 선수 이름을 목록 형태로 표시하도록 하였다.

---

### 자동 추천 기능

사용자가 선수 이름 전체를 입력하지 않더라도 일부만 입력하면 관련 선수 후보를 표시하도록 설계하였다.

예를 들어 사용자가 다음과 같이 입력할 경우

```text
호날
```

관련 선수 후보를 자동으로 표시하여 사용자가 원하는 선수를 쉽게 찾을 수 있도록 구현하기로 하였다.

---

### 국적 검색

사용자가 특정 국가를 검색하면 해당 국가 국적을 가진 선수 목록을 조회할 수 있도록 설계하였다.

예를 들어

```text
대한민국
```

을 입력하면 대한민국 국적 선수 목록을 표시하도록 구현하기로 하였다.
이때 국가 이름을 다 검색하지 않아도 뜨도록 하였다. 

---

### 클럽 검색

사용자가 클럽 이름을 검색하면 해당 클럽 정보를 조회할 수 있도록 설계하였다.

예를 들어

```text
Real Madrid
```

를 검색하면 레알 마드리드 관련 정보를 표시하도록 구성하였다.
이때도 이름을 다 입력하지 않아도 후보가 뜨도록 하였다. 

---

## 클럽 목록 기능 설계

논의 과정에서 사용자마다 클럽을 부르는 명칭이 다르며 정확한 클럽명을 모를 수도 있다는 문제를 확인하였다.

이를 해결하기 위해 별도의 "클럽 목록" 기능을 추가하기로 결정하였다.

사용자가 클럽 목록 버튼을 클릭하면 DB에 저장된 모든 클럽을 확인할 수 있도록 설계하였다.

---

## 클럽 선택 기반 선수 조회 기능

클럽 목록에서 특정 클럽을 선택하면 해당 클럽에서 활동했던 선수 목록을 조회할 수 있도록 설계하였다.

예를 들어 사용자가

```text
Real Madrid
```

를 선택하면 해당 클럽 경력을 가진 선수 목록이 표시되도록 구현하기로 하였다.

이를 통해 사용자가 직접 클럽명을 입력하지 않더라도 원하는 선수 정보를 찾을 수 있도록 하였다.

---

## 한글 선수 우선 표시 정책

현재 DB에는 한글 이름이 존재하는 선수와 영문 이름만 존재하는 선수가 함께 저장되어 있다.

하지만 실제 서비스를 사용할 사용자 입장에서는 너무 많은 검색 결과가 표시될 경우 원하는 선수를 찾기 어려울 수 있다고 판단하였다.

따라서 백엔드에서 이미 구현한 `has_korean_name` 필드를 활용하여 한글 이름이 존재하는 선수만 검색 결과에 표시하도록 설계하였다.

이를 통해 검색 결과를 단순화하고 사용자 편의성을 향상시키고자 하였다.
