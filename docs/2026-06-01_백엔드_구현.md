# 2026-06-01 실제 백엔드 구현

## FastAPI 기반 백엔드 구현

이전 단계에서 설계한 백엔드 구조를 바탕으로 실제 API 서버를 구현하였다.  
백엔드는 Python의 FastAPI를 사용하여 작성하였으며, 파일 이름은 `main.py`로 구성하였다.

이번 구현의 핵심 목적은 사용자가 프론트엔드에서 검색어를 입력했을 때, 백엔드가 SQLite 데이터베이스인 `football_players.db`를 조회하여 선수, 클럽, 국적 정보를 반환하도록 하는 것이었다.

---

## 사용 기술

백엔드 구현에는 다음 기술을 사용하였다.

- Python
- FastAPI
- SQLite
- pathlib

## DB 연결 설정

```python
DB_PATH = Path(__file__).resolve().parent.parent / "data" / "football_players.db"
```

백엔드에서 사용할 SQLite DB 파일의 경로를 설정하였다.  
`main.py` 파일을 기준으로 상위 폴더의 `data` 폴더 안에 있는 `football_players.db` 파일을 불러오도록 구성하였다.

```python
def get_db_connection():
    conn = sqlite3.connect(DB_PATH)
    conn.row_factory = sqlite3.Row
    return conn
```

DB 연결을 반복적으로 사용하기 위해 `get_db_connection()` 함수를 따로 작성하였다.  
`row_factory = sqlite3.Row`를 설정하여 SQL 조회 결과를 딕셔너리처럼 컬럼명으로 접근할 수 있도록 하였다.

## 선수 검색 API 구현

```python
@app.get("/search/player")
def search_player(q: str = Query(..., description="검색할 선수 이름 또는 별칭")):
```

사용자가 선수 이름 또는 별칭을 입력했을 때 관련 선수를 검색하는 API를 구현하였다.

검색 대상은 다음과 같다.

- `display_name`
- `wikidata_name`
- `alias`

```sql
WHERE p.has_korean_name = 1
  AND (
      p.display_name LIKE ?
      OR p.wikidata_name LIKE ?
      OR pa.alias LIKE ?
  )
```

검색 결과에서는 `has_korean_name = 1`인 선수만 조회하도록 하였다.  
이를 통해 영문 이름만 존재하는 선수는 기본 검색 결과에서 제외하고, 한글 이름이 존재하는 선수만 사용자에게 보여주도록 구현하였다.

또한 `LIKE` 검색을 사용하여 사용자가 이름의 일부만 입력해도 검색 결과가 나오도록 하였다.  
예를 들어 사용자가 `호날`만 입력하더라도 `호날두` 관련 선수가 검색될 수 있다.

## 선수 상세 정보 API 구현

```python
@app.get("/players/{qid}")
def get_player_detail(qid: str):
```

검색 결과에서 특정 선수를 선택했을 때, 해당 선수의 상세 정보를 조회하는 API를 구현하였다.

이 API는 선수의 `qid`를 기준으로 다음 정보를 조회한다.

- 선수 기본 정보
- 국적 정보
- 클럽 경력 정보

```sql
SELECT DISTINCT country
FROM player_countries
WHERE qid = ?
```

선수의 국적 정보는 `player_countries` 테이블에서 조회하였다.  
복수국적 선수가 존재할 수 있기 때문에 `DISTINCT`를 사용하여 중복을 제거하였다.

```sql
SELECT DISTINCT c.club_qid, c.club_name
FROM player_clubs pc
JOIN clubs c
ON pc.club_qid = c.club_qid
WHERE pc.qid = ?
```

선수의 클럽 경력은 `player_clubs` 테이블과 `clubs` 테이블을 JOIN하여 조회하였다.  
이를 통해 선수 QID를 기준으로 해당 선수가 활동했던 클럽 목록을 반환할 수 있도록 구현하였다.

## 클럽 검색 API 구현

```python
@app.get("/search/club")

```

사용자가 클럽 이름을 입력했을 때 관련 클럽을 검색하는 API를 구현하였다.

```sql
SELECT club_qid, club_name
FROM clubs
WHERE club_name LIKE ?
ORDER BY club_name
LIMIT 30
```

클럽명 역시 `LIKE` 검색을 사용하여 일부만 입력해도 검색이 가능하도록 하였다.  
예를 들어 `Real`을 입력하면 `Real Madrid`와 같은 클럽이 검색될 수 있다.

## 특정 클럽 소속 선수 조회 API 구현

```python
@app.get("/club/{club_qid}")
def get_players_by_club(club_qid: str):
```

사용자가 클럽 목록에서 특정 클럽을 선택했을 때, 해당 클럽에서 활동했던 선수 목록을 조회하는 API를 구현하였다.

```sql
SELECT DISTINCT
    p.qid,
    p.display_name,
    p.wikidata_name
FROM players p
JOIN player_clubs pc
ON p.qid = pc.qid
WHERE pc.club_qid = ?
  AND p.has_korean_name = 1
ORDER BY p.display_name
```

이 API에서도 `has_korean_name = 1` 조건을 적용하였다.  
따라서 클럽에 소속된 모든 선수 중 한글 이름이 존재하는 선수만 사용자에게 보여주도록 하였다.

이를 통해 사용자가 클럽명을 직접 검색하지 않고도, 클럽 목록에서 특정 클럽을 클릭하여 관련 선수들을 확인할 수 있도록 구현하였다.

## 국적 검색 API 구현

```python
@app.get("/search/country")
def search_country(q: str):
```

사용자가 특정 국가를 입력했을 때 해당 국적을 가진 선수들을 검색하는 API를 구현하였다.

```sql
SELECT DISTINCT
    p.qid,
    p.display_name,
    p.wikidata_name,
    pc.country
FROM players p
JOIN player_countries pc
ON p.qid = pc.qid
WHERE p.has_korean_name = 1
  AND pc.country LIKE ?
ORDER BY p.display_name
LIMIT 100
```

국적 정보는 `player_countries` 테이블과 `players` 테이블을 JOIN하여 조회하였다.  
복수국적 선수도 검색될 수 있도록 국적 정보를 별도 테이블에서 조회하였다.

예를 들어 사용자가 `대한민국`을 입력하면 대한민국 국적을 가진 선수들이 검색되도록 구현하였다.
더하여, `대한` 만 검색해도 대한민국 국적의 선수가 출력이 되도록 하였다.
