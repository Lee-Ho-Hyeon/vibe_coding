# 2026-05-22 설계 개선 및 데이터 수집 방식 재논의


## 데이터 수집 방식 변경 논의

문제 해결을 위해 데이터 수집 구조를 다시 논의하였다. 논의 결과, 서로 다른 사이트를 조합하여 사용하는 방식보다는 하나의 사이트에서 선수 이름과 클럽 경력을 함께 가져오는 방식이 더 안정적이라고 판단하였다.  

이에 따라 무료 API를 제공하고 있는 위키데이터(Wikidata)를 중심으로 데이터 수집 구조를 변경하기로 결정하였다. 위키데이터는 선수 정보와 클럽 경력 정보가 함께 연결되어 있어 데이터 일관성을 유지하기에 적합하다고 판단하였다.  

하지만 여기서 또 다른 문제가 발생하였다. 위키데이터에 존재하는 모든 축구 선수를 자동으로 탐색하여 저장하는 것은 사실상 불가능하다는 점이었다. 선수 수가 매우 많고, 모든 선수를 하나씩 직접 검색하여 저장하는 방식은 현실적으로 구현 및 관리 비용이 지나치게 크다고 판단하였다.

## 클럽 기반 데이터 수집 방식으로 변경

이에 따라 데이터 수집 방식을 다시 수정하였다. 논의 결과, 위키데이터에서는 특정 선수 검색 시 해당 선수의 클럽 경력을 확인할 수 있을 뿐만 아니라, 반대로 특정 클럽을 검색하면 해당 클럽에서 활동했던 선수 목록 또한 제공된다는 점을 확인하였다.  

따라서 “모든 선수를 직접 찾는 방식” 대신, “특정 클럽에 소속되었던 선수들을 모두 불러와 저장하는 방식”으로 방향을 변경하였다.  

예를 들어 유명 클럽을 기준으로 선수 데이터를 수집하면 비교적 안정적으로 많은 선수 정보를 확보할 수 있으며, 데이터 수집 범위를 관리하기에도 용이하다고 판단하였다.  

이후 이렇게 클럽별로 선수들을 모은 후 추후에 선수로 검색을 하면 클럽 경력이 뜨도록 DB를 만들 예정이다.

## 선수 이름 검색 편의성 문제 논의

추가적으로 선수 이름 검색 과정에서도 문제가 발견되었다. 위키데이터에 저장된 선수 이름은 실제 게임이나 일반 사용자들이 자주 사용하는 이름과 다른 경우가 많았으며, 일부 선수는 전체 이름(full name)을 정확히 입력하지 않으면 검색되지 않는 문제가 존재하였다.  

예를 들어 실제 사용자들은 “크리스티아누 호날두”와 같이 일반적으로 알려진 이름을 사용하지만, 데이터에는 전체 이름이나 다른 표기 방식으로 저장되어 있는 경우가 존재하였다.  

이에 따라 논의 결과, 선수의 전체 이름만 저장하는 것이 아니라 별칭(alias)이나 일반적으로 많이 사용되는 이름 또한 함께 저장하기로 결정하였다. 이를 통해 사용자가 실제로 검색할 가능성이 높은 이름 기반으로 검색 편의성을 향상시키고자 하였다.
