# 2025-06-03_프론트엔드_수정

## 페이지 분리 방식으로 구조 수정

초기 프론트엔드는 모든 기능을 하나의 `index.html` 화면 안에서 처리하도록 구현하였다.  
검색 결과, 클럽 목록, 국적별 선수 목록, 선수 상세 정보가 모두 같은 페이지에 표시되었기 때문에 기능 자체는 정상적으로 동작하였으나, 검색 결과가 많아질수록 화면이 길어지고 사용자가 원하는 정보를 찾기 어려운 문제가 발생하였다.

특히 클럽 검색과 국적 검색의 경우 한 번에 출력되는 선수 수가 많기 때문에, 하나의 화면 안에서 모든 결과를 표시하는 방식은 가독성이 떨어진다고 판단하였다.

이에 따라 프론트엔드 구조를 다음과 같이 여러 페이지로 분리하였다.

```text
index.html
- 검색 메인 페이지

club.html
- 특정 클럽의 선수 목록 페이지

country.html
- 특정 국적의 선수 목록 페이지

club-list.html
- 전체 클럽 목록 페이지

player.html
- 선수 상세 정보 페이지
```

이를 통해 사용자는 검색 결과를 선택하면 별도의 페이지로 이동하여 관련 정보를 확인할 수 있도록 개선하였다.

---

## player.html 및 player.js 구현

먼저 선수 상세 정보를 별도 페이지에서 확인할 수 있도록 `player.html`과 `player.js`를 구현하였다.

기존에는 검색 결과에서 선수를 클릭하면 같은 화면 아래쪽에 선수 상세 정보가 출력되었다. 그러나 이 방식은 검색 결과와 상세 정보가 한 화면에 섞여 보이는 문제가 있었다.

따라서 선수를 클릭하면 다음과 같은 주소로 이동하도록 수정하였다.

```text
player.html?qid=Q11571
```

`player.js`에서는 URL에 포함된 `qid` 값을 읽어 백엔드의 선수 상세 조회 API를 호출하도록 구현하였다.

```text
/players/{qid}
```

이를 통해 선수 상세 페이지에서는 다음 정보를 독립적으로 표시할 수 있도록 하였다.

- 선수 이름
- 국적
- 클럽 경력

예를 들어 사용자가 검색 결과에서 크리스티아누 호날두를 선택하면 `player.html?qid=Q11571` 페이지로 이동하고, 해당 페이지에서 국적과 클럽 경력을 확인할 수 있도록 구현하였다.

---

## club.html 및 club.js 구현

다음으로 클럽 검색 결과를 별도 페이지에서 확인할 수 있도록 `club.html`과 `club.js`를 구현하였다.

기존에는 클럽명을 검색한 뒤 특정 클럽을 선택하면 같은 화면 아래쪽에 해당 클럽에서 활동한 선수 목록이 출력되었다. 하지만 클럽 경력을 가진 선수 수가 많을 경우 검색 화면이 지나치게 길어지는 문제가 있었다.

이를 해결하기 위해 클럽 후보를 선택하면 다음과 같은 주소로 이동하도록 수정하였다.

```text
club.html?club_qid=Q8682&club_name=Real%20Madrid
```

`club.js`에서는 URL에 포함된 `club_qid`를 읽어 백엔드의 클럽별 선수 조회 API를 호출하도록 구현하였다.

```text
/club/{club_qid}
```

이를 통해 특정 클럽에서 활동했던 선수 목록을 별도 페이지에서 확인할 수 있게 되었다.

또한 클럽 선수 목록에서 특정 선수를 선택하면 다시 `player.html?qid=선수QID` 형식으로 이동하여 선수 상세 정보를 확인할 수 있도록 연결하였다.

---

## club-list.html 및 club-list.js 구현

클럽 목록 조회 기능도 별도 페이지로 분리하였다.

기존에는 `index.html`에서 "클럽 목록 보기" 버튼을 클릭하면 같은 화면 아래쪽에 모든 클럽 목록이 출력되었다. 그러나 클럽 수가 많아질 경우 메인 화면의 가독성이 떨어지는 문제가 있었다.

이에 따라 `club-list.html`과 `club-list.js`를 새로 구현하였다.

사용자가 "클럽 목록 보기" 버튼을 클릭하면 다음 페이지로 이동하도록 수정하였다.

```text
club-list.html
```

`club-list.js`에서는 백엔드의 전체 클럽 목록 API를 호출하도록 구현하였다.

```text
/clubs
```

이후 사용자가 클럽 목록에서 특정 클럽을 선택하면 `club.html?club_qid=클럽QID&club_name=클럽명` 형식으로 이동하여 해당 클럽의 선수 목록을 확인할 수 있도록 하였다.

이를 통해 클럽 목록 조회 기능 역시 메인 화면과 분리되어 더 깔끔하게 사용할 수 있게 되었다.

---

## country.html 및 country.js 구현

국적 검색 기능도 클럽 검색과 동일한 방식으로 별도 페이지로 분리하였다.

기존에는 사용자가 국적을 검색하고 특정 국가를 선택하면 같은 화면에 해당 국적 선수 목록이 바로 표시되었다. 그러나 특정 국적에 포함된 선수가 많을 경우 검색 화면이 지나치게 길어지는 문제가 있었다.

따라서 국적 후보를 선택하면 다음과 같은 주소로 이동하도록 수정하였다.

```text
country.html?country=이탈리아
```

`country.js`에서는 URL에 포함된 `country` 값을 읽어 백엔드의 국적별 선수 검색 API를 호출하도록 구현하였다.

```text
/search/country?q=이탈리아
```

이를 통해 사용자는 특정 국적 선수 목록을 별도 페이지에서 확인할 수 있게 되었으며, 선수 목록에서 특정 선수를 선택하면 `player.html?qid=선수QID` 페이지로 이동하여 상세 정보를 확인할 수 있도록 구현하였다.

## app.js 이동 방식 수정

페이지 분리 구조에 맞게 `app.js`의 동작 방식도 수정하였다.

기존에는 검색 결과를 클릭하면 같은 화면 안에서 선수 목록이나 상세 정보를 출력하였다. 수정 후에는 검색 결과를 클릭했을 때 각 기능에 맞는 페이지로 이동하도록 변경하였다.

수정된 흐름은 다음과 같다.

```text
선수 검색 결과 클릭
→ player.html?qid=선수QID

클럽 검색 결과 클릭
→ club.html?club_qid=클럽QID&club_name=클럽명

국적 검색 결과 클릭
→ country.html?country=국가명

클럽 목록 보기 클릭
→ club-list.html
```

이를 통해 메인 페이지는 검색과 후보 표시 역할에 집중하고, 실제 목록 및 상세 정보는 별도 페이지에서 확인하는 구조로 개선하였다.

## 홈 버튼 추가

페이지가 여러 개로 분리되면서 사용자가 언제든지 메인 검색 화면으로 돌아갈 수 있는 기능이 필요하다고 판단하였다.

이에 따라 모든 페이지에 홈 버튼을 추가하였다.

```html
<a href="index.html" class="home-button">홈</a>
```

홈 버튼은 `style.css`에서 고정 위치로 배치하였으며, 사용자가 어느 페이지에 있더라도 오른쪽 위에서 쉽게 확인할 수 있도록 하였다.

이를 통해 다음 페이지들에서 모두 메인 화면으로 돌아갈 수 있도록 개선하였다.

- `index.html`
- `club.html`
- `country.html`
- `club-list.html`
- `player.html`

## 수정 후 최종 프론트엔드 구조

페이지 분리 이후 최종 프론트엔드 파일 구조는 다음과 같이 구성되었다.

```text
frontend
├── index.html
├── club.html
├── country.html
├── club-list.html
├── player.html
│
├── app.js
├── club.js
├── country.js
├── club-list.js
├── player.js
│
└── style.css
```

각 파일의 역할은 다음과 같다.

- `index.html`
  - 메인 검색 화면
- `app.js`
  - 검색창, 자동 추천, 검색 결과 클릭 시 페이지 이동 처리
- `club.html`, `club.js`
  - 특정 클럽의 선수 목록 표시
- `country.html`, `country.js`
  - 특정 국적의 선수 목록 표시
- `club-list.html`, `club-list.js`
  - 전체 클럽 목록 표시
- `player.html`, `player.js`
  - 선수 상세 정보 표시
- `style.css`
  - 전체 화면 디자인 및 홈 버튼 스타일 관리

## 수정 결과

페이지 분리 이후 검색 결과와 상세 정보가 한 화면에 모두 표시되는 문제가 개선되었다.

사용자는 메인 화면에서 검색어를 입력하고 후보를 선택한 뒤, 별도의 페이지에서 목록 또는 상세 정보를 확인할 수 있게 되었다.

최종적으로 다음 흐름이 가능해졌다.

```text
선수 검색
→ 선수 선택
→ 선수 상세 정보 페이지 이동

클럽 검색
→ 클럽 선택
→ 클럽 선수 목록 페이지 이동
→ 선수 선택
→ 선수 상세 정보 페이지 이동

국적 검색
→ 국적 선택
→ 국적 선수 목록 페이지 이동
→ 선수 선택
→ 선수 상세 정보 페이지 이동

클럽 목록 보기
→ 전체 클럽 목록 페이지 이동
→ 클럽 선택
→ 클럽 선수 목록 페이지 이동
```
