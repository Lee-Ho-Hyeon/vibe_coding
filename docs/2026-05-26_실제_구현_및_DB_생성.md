# 2026-05-26 실제 구현 및 DB 생성

## CSV 데이터를 DB로 변환하는 기능 구현

이전 설계 및 논의 내용을 바탕으로, CSV 형태로 저장된 선수 데이터를 실제 데이터베이스(DB)에 저장하는 작업을 진행하였다.  

구현 과정에서는 다음 두 종류의 CSV 파일을 입력 데이터로 사용하였다.

- `club_qid_list.csv`
- `club_{club_qid}_players.csv`

`club_qid_list.csv`에는 수집 대상 클럽들의 고유 식별자(club_qid) 정보가 저장되어 있었으며, 각 `club_{club_qid}_players.csv` 파일에는 특정 클럽에 소속되었던 선수들의 정보가 저장되어 있었다.  

이를 기반으로 CSV 데이터를 자동으로 읽어 DB에 저장하는 `csv_to_db.py` 파일을 구현하였다.  

---

## SQLite 기반 DB 생성

데이터베이스 시스템은 SQLite를 사용하기로 결정하였다. SQLite는 별도의 서버 구축 없이 하나의 파일 형태로 DB를 관리할 수 있기 때문에 현재 프로젝트 규모에 적합하다고 판단하였다.  

`csv_to_db.py` 실행 결과 최종적으로 `football_players.db` 파일을 생성하였다.  

또한 이전 설계 단계에서 논의한 구조를 기반으로 선수 정보, 클럽 정보, alias 정보, 국적 정보 등을 각각의 테이블에 분리하여 저장하도록 구현하였다.  

---

## 중복 선수 데이터 처리 문제

구현 과정에서 추가적인 문제점도 확인되었다. 하나의 선수가 여러 클럽에서 활동한 경력이 있을 경우, 클럽별 CSV 파일에 동일 선수가 반복적으로 저장되는 문제가 존재하였다.  

예를 들어 한 선수가 맨체스터 유나이티드와 레알 마드리드 모두에서 활동한 경우, 각 클럽 CSV 파일에 동일 선수 정보가 모두 포함되어 있었다.  

따라서 DB 저장 과정에서는 동일 선수 데이터가 중복 저장되지 않도록 처리하는 로직이 필요하였다. 이를 위해 선수의 고유 식별자인 `qid`를 기준으로 이미 저장된 선수인지 확인한 뒤, 중복 선수 정보는 한 번만 저장하도록 구현하였다.  

이를 통해 데이터 중복 문제를 줄이고 DB 구조의 일관성을 유지할 수 있도록 구현을 진행하였다.
