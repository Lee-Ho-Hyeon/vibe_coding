# 2026-05-31 데이터 확장 및 검색 대상 선수 선별

## 데이터 수집 범위 확장

기존 구현 단계에서는 일부 유명 클럽을 대상으로 선수 정보를 수집하여 데이터베이스를 구축하였다. 이러한 방식은 구현이 간단하다는 장점이 있었지만, 특정 클럽에 소속되었던 선수들만 검색할 수 있다는 한계가 존재하였다.

이에 따라 데이터의 범위를 확대하기 위해 유럽 Top 5 리그에 소속된 모든 팀을 대상으로 선수 정보를 수집하기로 결정하였다. 이후 각 클럽의 선수 데이터를 수집하여 기존 DB를 확장하였으며, 보다 많은 선수 정보를 검색할 수 있도록 개선하였다.

---

## 데이터 증가에 따른 문제점 발견

데이터 수집 범위를 확장한 결과 선수 데이터 수가 크게 증가하였다. 그러나 실제 데이터를 확인해본 결과, 일부 클럽에는 일반 사용자들에게 잘 알려지지 않은 선수들까지 대량으로 포함되어 있었다.

이로 인해 사용자가 특정 선수를 검색할 경우 지나치게 많은 결과가 반환될 가능성이 있었으며, 원하는 선수를 찾기 어려워질 수 있다는 문제가 발견되었다.

특히 위키데이터에는 존재하지만 실제 게임에서는 등장하지 않거나 일반 사용자들이 거의 검색하지 않을 것으로 예상되는 선수들도 다수 포함되어 있었다.

---

## 검색 대상 선수 선별 기준 논의

문제를 해결하기 위해 수집된 선수들을 일정 기준으로 분류하는 방안을 논의하였다.

현재 저장된 데이터에는 다음과 같은 정보가 존재한다.

- wikidata_name
- display_name
- alias

논의 결과, 위 세 항목 중 하나라도 한글 이름이 존재하는 선수와 영문 이름만 존재하는 선수를 구분하기로 결정하였다.

그 이유는 한글 이름이 존재하는 선수의 경우 국내 사용자들에게 어느 정도 알려져 있거나 실제 게임 내에서도 사용될 가능성이 높은 선수들이라고 판단하였기 때문이다.

반면, 모든 이름 정보가 영문으로만 존재하는 선수들은 상대적으로 인지도가 낮거나 실제 게임에서 활용되지 않을 가능성이 높다고 판단하였다.

---

## 데이터 분류 방식 적용

이에 따라 선수 데이터를 다음 두 그룹으로 구분하도록 수정하였다.

### 한글 이름이 존재하는 선수

- wikidata_name, display_name, alias 중 하나 이상에 한글 이름 존재
- 검색 대상 선수로 사용

### 영문 이름만 존재하는 선수

- 모든 이름 정보가 영문으로만 구성
- 별도 분류하여 저장

이러한 분류를 통해 데이터는 유지하되, 실제 검색 결과에서는 사용자 편의성을 높일 수 있도록 설계하였다.

---

## 향후 적용 계획

현재 단계에서는 선수 데이터를 분류하는 기능까지 구현하였다.

향후 검색 기능을 구현할 때에는 영문 이름만 존재하는 선수들은 기본 검색 결과에서 제외하고, 한글 이름이 존재하는 선수들을 우선적으로 제공하도록 구현할 예정이다.
