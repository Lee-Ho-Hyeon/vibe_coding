# 2026-06-01 백엔드 개발 및 검색 기능 설계

## 백엔드 개발 필요성

이전 단계에서 선수 정보를 SQLite 데이터베이스인 `football_players.db`에 저장하는 작업을 완료하였다. 또한 데이터 수집 범위를 기존의 일부 유명 클럽에서 유럽 Top 5 리그 전체 팀으로 확대하여 보다 많은 선수 정보를 확보하였다.

하지만 현재 상태에서는 데이터가 DB에 저장되어 있을 뿐, 사용자가 원하는 선수를 검색하거나 클럽 및 국적 정보를 조회할 수 있는 기능이 존재하지 않았다.

따라서 사용자의 검색 요청을 받아 데이터베이스를 조회하고 결과를 반환하는 백엔드 시스템을 구현하기로 하였다.

---

## 백엔드 구조 설계

논의 결과 백엔드는 데이터베이스와 사용자 화면 사이에서 중간 역할을 수행하도록 설계하였다.

사용자가 선수 이름, 별칭(alias), 국적 또는 클럽명을 입력하면 백엔드가 이를 받아 DB를 조회하고, 조회 결과를 프론트엔드에 전달하는 구조로 결정하였다.

이를 통해 데이터베이스에 직접 접근하지 않고도 원하는 정보를 검색할 수 있도록 하였다.

---

## 검색 대상 선수 선별

데이터 수집 범위가 크게 증가하면서 사용자들이 실제로 검색하지 않을 가능성이 높은 선수들까지 대량으로 포함되었다.

이에 따라 이전 단계에서 `has_korean_name` 필드를 추가하여 선수를 분류하였다.

- 한글 이름이 존재하는 선수 : `has_korean_name = 1`
- 영문 이름만 존재하는 선수 : `has_korean_name = 0`

실제 검색 결과에서는 `has_korean_name = 1`인 선수들만 우선적으로 제공하도록 설계하였다.

이를 통해 검색 결과 수를 줄이고 사용자 편의성을 향상시키고자 하였다.

---

## 개발 환경 선정

백엔드 프레임워크는 Python 기반의 FastAPI를 사용하기로 결정하였다.

FastAPI는 비교적 적은 코드로 API 서버를 구축할 수 있으며, SQLite와의 연동이 쉽고 현재 프로젝트 규모에 적합하다고 판단하였다.

사용 기술은 다음과 같다.

- Python
- FastAPI
- SQLite
- Uvicorn

## 검색 기능 구현 계획

백엔드에서는 다음과 같은 검색 기능을 제공하도록 설계하였다.

### 선수 이름 및 Alias 검색

사용자가 선수 이름 또는 별칭(alias)을 입력하면 해당 선수 정보를 조회하도록 설계하였다.

예를 들어 사용자가 "Cristiano Ronaldo"뿐만 아니라 "호날두", "CR7" 등을 입력하더라도 동일 선수 정보를 찾을 수 있도록 구현하기로 하였다.

---

### 자동 추천 검색 기능

사용자가 선수 이름 전체를 입력하지 않아도 일부만 입력하면 관련 선수 후보를 추천하도록 설계하였다.

예를 들어 사용자가 "호날"만 입력하더라도 "크리스티아누 호날두"와 같은 선수 후보를 화면에 표시할 수 있도록 구현하기로 하였다.

이를 위해 부분 문자열 검색 기능을 활용하기로 결정하였다.

---

### 국적 검색

사용자가 특정 국가를 입력하면 해당 국적을 가진 선수 목록을 조회할 수 있도록 설계하였다.

예를 들어 "대한민국"을 검색하면 대한민국 국적을 가진 선수들을 반환하도록 구현하기로 하였다.

---

### 클럽 검색

사용자가 특정 클럽명을 입력하면 해당 클럽에서 활동했던 선수 목록을 조회할 수 있도록 설계하였다.

예를 들어 "Real Madrid"를 검색하면 레알 마드리드에서 활동한 선수들을 반환하도록 구현하기로 하였다.

---

### 클럽 목록 조회 기능

사용자마다 동일한 클럽을 부르는 이름이 다를 수 있으며, 정확한 클럽명을 모르는 경우도 존재하였다.

이를 해결하기 위해 별도의 "클럽 목록" 기능을 제공하기로 하였다.

사용자가 클럽 목록 메뉴를 선택하면 DB에 저장된 모든 클럽을 확인할 수 있도록 설계하였다.

---

### 클럽 선택 기반 검색

클럽 목록에서 특정 클럽을 선택하면 해당 클럽에서 활동한 선수 목록을 바로 조회할 수 있도록 설계하였다.

예를 들어 사용자가 "Real Madrid"를 클릭하면 레알 마드리드에서 활동한 선수들의 목록을 표시하도록 구현하기로 하였다.

이를 통해 클럽명을 직접 입력하지 않아도 원하는 선수 정보를 찾을 수 있도록 하였다.

---

## API 구조 설계

초기 API 구조는 다음과 같은 형태로 설계하였다.

- `/search/player`
- `/search/country`
- `/search/club`
- `/clubs`
- `/club/{club_qid}`

각 API는 사용자가 입력한 검색어를 전달받아 DB를 조회하고 결과를 JSON 형태로 반환하도록 구현하기로 하였다.
